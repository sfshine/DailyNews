## 1. 移动大前端

### 1.1 Android：从 OS 走向 Intelligence System

- Android Developers Blog 在 2026-05-12 提出 Android 正从操作系统转向“智能系统”，重点是 Gemini Intelligence 与跨应用任务自动化能力。对应用团队而言，未来竞争点不只是“用户打开 App”，还包括 App 是否能被 Gemini/Agent 正确调用并完成任务。
- 2026-02-25 的 Android AppFunctions 方向值得重点跟进：应用可以在本地把数据和功能暴露给 AI agents/assistants，类似移动端的 tool/function calling 入口。
- 2026-05-19，Google 宣布 Google AI Studio 可从 prompt 生成 Kotlin + Jetpack Compose 原生 Android App，并支持安装测试或转入 Android Studio 继续开发。
- Android Studio I/O Edition 强调从 AI-assisted 转向 Agentic development，Android Studio Canary 已捆绑 Android/Firebase agent skills。

**研判：** 移动端大前端的“入口战争”会从 UI/路由扩展到 Agent 可发现、可调用、可授权的函数层。Android 团队应提前定义哪些业务能力适合暴露给 AppFunctions，以及如何做权限、审计和回滚。

### 1.2 Flutter / React Native：版本迁移信号

- Flutter 官方 release notes 页面显示当前文档对应 Flutter 3.44.0，页面最后更新于 2026-05-15；Flutter docs 的 “What’s new” 标注 2026-05-18 Google I/O Release 3.44。
- React Native 官方版本表显示 0.86.x 分支切出时间为 2026-05-04，计划发布时间为 2026-06-08；0.85.x 仍为 Active。
- React Native 0.85 的重点包括新 animation backend、Jest preset 独立为 `@react-native/jest-preset`、Node.js 版本要求提升以及部分废弃 API 移除。

**研判：** Flutter 侧关注 3.44 的 Dart/Flutter 新能力与 breaking changes；RN 侧本周应关注 0.86 stable 是否按计划发布，并提前排查 Jest preset、Node.js、`StyleSheet.absoluteFillObject` 等迁移点。

**来源：**

- Flutter docs: What’s new, https://docs.flutter.dev/release/whats-new
- React Native versions, https://reactnative.dev/versions

## 2. AI Coding

### 2.1 Agentic coding 成为 Build / IDE 主线

- Microsoft Build 2026 相关报道继续把 GitHub Copilot、Azure AI Foundry、MCP、多智能体编排、VS Code 工作流作为开发者工具主线。
- Windows on Arm/RTX Spark 相关 Build session 也出现“用 AI agents 帮开发者移植 x86 应用到 Snapdragon X/RTX Spark”的信号，说明 coding agent 正从生成代码扩展到迁移、适配和重复性工程任务。
- JetBrains Central、IBM Bob、xAI Grok Build 等信息显示：AI Coding 正从单一 IDE 补全，转向“多 Agent、可治理、可编排、跨工具”的平台层能力。

**研判：** 团队采用 AI Coding 不应只比较补全质量，还要比较 Agent 权限模型、任务编排、MCP/工具接入、日志审计、企业代码保护和移动端审批体验。

### 2.2 Codex 移动化：审批与任务启动入口下沉到手机

- Axios 报道 OpenAI Codex 进入 ChatGPT mobile app，移动端可审查输出、批准任务并启动新任务。

**研判：** AI Coding 的协作链路会从“开发者坐在 IDE 前”变成“异步分派任务 + 移动端审核/批准 + 回到仓库合并”。这对代码评审策略、CI 质量门禁和权限边界提出更高要求。

## 3. 具身人工智能

### 3.1 ICRA 2026：灵巧手与通用物理智能演示升温

- TARS 在 ICRA 2026 展示 A1 机器人、DexHand 与 AWE 3.0，演示多步打包、拉链、亚毫米线束插入和实时纠错等复杂操作。
- RobotD 近期聚合信息显示，ICRA 2026 周期出现自行车机器人前空翻、受损六足机器人 11 分钟学会新步态、NVIDIA Cosmos 3 Physical AI foundation model、BYD 确认 humanoid robotics division 等信号。

**研判：** 具身智能的热点正在从“能不能动”转向“能不能泛化地操作物体、在错误中恢复、在真实硬件上快速适应”。但大部分信息仍需区分实验演示、论文指标和可复制部署。

### 3.2 落地瓶颈：可靠性、安全、成本、数据和治理

- Computer Weekly 对 ATxSummit 2026 的报道指出，具身 AI 正走出实验室，但规模化采用仍受可靠性、安全、成本、数据可得性和治理标准限制。
- 日本 AIRoA 目标收集 100,000 小时机器人数据的开放数据倡议，说明“机器人基础模型”竞争并不只是模型结构，还包括高质量真实世界数据供给。

**研判：** 对具身 AI 项目，应重点追踪数据闭环、仿真到现实迁移、硬件 BOM、失效恢复、安全认证，而不是只看视频 demo。
